# Firebase Auth App (Flutter + Firebase Authentication + Cloud Firestore)

A Flutter application demonstrating:
- Email/Password authentication with Firebase Authentication
- Login and Signup screens
- Automatic navigation to a Profile screen after login
- Storing and retrieving user details (name, email) in Cloud Firestore

## Project Structure

```
lib/
├── main.dart                  # App entry point + AuthGate (routes based on login state)
├── firebase_options.dart      # Generated by FlutterFire CLI (placeholder here)
├── models/
│   └── user_model.dart        # UserModel: maps to/from Firestore documents
├── services/
│   ├── auth_service.dart      # All FirebaseAuth calls (sign up/in/out)
│   └── firestore_service.dart # All Firestore reads/writes
└── screens/
    ├── login_screen.dart
    ├── signup_screen.dart
    └── profile_screen.dart
```

## How It Works

1. **`main.dart`** initializes Firebase, then shows an `AuthGate` widget.
   `AuthGate` listens to `FirebaseAuth.instance.authStateChanges()` — a
   stream that fires whenever a user logs in or out. Based on that stream,
   it automatically shows `LoginScreen` (signed out) or `ProfileScreen`
   (signed in). This means you never have to manually navigate after
   login — it happens automatically.

2. **`SignupScreen`** collects name, email, and password. It:
   - Calls `AuthService.signUp()` to create the Firebase Auth account.
   - Then calls `FirestoreService.saveUserData()` to save the name and
     email into a `users` collection in Firestore, keyed by the new
     user's UID.

3. **`LoginScreen`** collects email and password and calls
   `AuthService.signIn()`.

4. **`ProfileScreen`** reads the currently signed-in user's UID and uses
   `FirestoreService.streamUserData()` to live-stream that user's saved
   name/email from Firestore and display it. It also has a sign-out button.

## Prerequisites

- Flutter SDK installed (`flutter --version` to check)
- A Google account to create a Firebase project
- Node.js (needed for the Firebase CLI) — https://nodejs.org
- Xcode (for iOS) and/or Android Studio (for Android), if targeting those platforms

## Setup Instructions (Step by Step)

### 1. Create the Flutter project (if not already created)
If you're starting fresh instead of using these files directly:
```bash
flutter create firebase_auth_app
cd firebase_auth_app
```
If you're using the files provided here, just place them into a fresh
`flutter create` project, replacing the generated `lib/main.dart` and
`pubspec.yaml`.

### 2. Create a Firebase Project
1. Go to https://console.firebase.google.com
2. Click **Add project**, name it (e.g. `firebase-auth-app`), and finish the wizard.

### 3. Enable Email/Password Authentication
1. In the Firebase Console, go to **Build → Authentication**.
2. Click **Get Started**.
3. Under the **Sign-in method** tab, enable **Email/Password**.

### 4. Create the Firestore Database
1. In the Firebase Console, go to **Build → Firestore Database**.
2. Click **Create database**.
3. Choose **Start in test mode** for development (see security rules note below).
4. Pick a region and finish.

### 5. Install the Firebase CLI and FlutterFire CLI
```bash
npm install -g firebase-tools
firebase login

dart pub global activate flutterfire_cli
```

### 6. Connect Your Flutter App to Firebase
From inside your Flutter project folder:
```bash
flutterfire configure
```
- Select (or create) your Firebase project.
- Select the platforms you want (Android, iOS, Web, etc.).
- This command automatically **overwrites** `lib/firebase_options.dart`
  with your real project keys and registers your app with Firebase
  (creating `google-services.json` for Android and/or
  `GoogleService-Info.plist` for iOS automatically).

> The `firebase_options.dart` included in this project is only a
> **placeholder** with dummy values — you must run `flutterfire configure`
> to generate the real one, or the app will fail to connect to Firebase.

### 7. Install Dependencies
```bash
flutter pub get
```

### 8. Run the App
```bash
flutter run
```

### 9. Test the Flow
1. Open the app → you'll land on the **Login** screen.
2. Tap **"Don't have an account? Sign up"**.
3. Enter a name, email, and password (6+ characters) → tap **Sign Up**.
4. You'll be redirected automatically to the **Profile** screen, showing
   your saved name and email pulled live from Firestore.
5. Tap the logout icon in the app bar → you're returned to the Login screen.
6. Log back in with the same email/password to confirm persistence.

## Firestore Data Model

Collection: `users`
```
users/
  {uid}/
    name: "Jane Doe"
    email: "jane@example.com"
    createdAt: "2026-07-07T10:00:00.000"
```

## Firestore Security Rules (Recommended for Production)

Test mode leaves the database open to anyone. Before shipping, set rules
like this in **Firestore → Rules**:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{uid} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
    }
  }
}
```
This ensures a user can only read/write their own document.

## Troubleshooting

| Problem | Likely Cause / Fix |
|---|---|
| `[core/no-app]` error on launch | `flutterfire configure` wasn't run, or `firebase_options.dart` still has placeholder values |
| Sign up fails silently | Check that Email/Password sign-in is enabled in Firebase Console |
| Profile screen shows no data | Confirm the `users/{uid}` document was created — check the Firestore console |
| Android build fails | Ensure `google-services.json` is in `android/app/` (created by `flutterfire configure`) |
| iOS build fails | Ensure `GoogleService-Info.plist` is in `ios/Runner/` (created by `flutterfire configure`) |
